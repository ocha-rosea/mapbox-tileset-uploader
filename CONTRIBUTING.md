# Contributing

Contributions are welcome.

## How to Contribute

1. Fork the repository.
2. Create a feature branch.
3. Make your changes with focused commits.
4. Run checks before opening a pull request:
   - `pytest`
   - `ruff check .`
5. Open a pull request with a clear summary of what changed and why.

## Development Notes

- Keep changes minimal and scoped to the issue.
- Update documentation when behavior changes.
- Prefer tests for changed behavior where practical.
